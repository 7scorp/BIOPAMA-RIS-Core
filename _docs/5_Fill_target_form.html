  • Add a name for your Target.
  • Add a link to the website where the Policy document can be read. Link text will provide more info on it (optional).
  • Add a description for your Target.
    • Specify the date by which the Target should be achieved, based on the information provided in the Policy document. Specify the priority for achieving this Targe. Priority actions are considered in relation to the Action Component. Don't forget to save!

DIFFERENCE BETWEEN TARGET AND INDICATOR:

An indicator measures progresses over time against a target value. A target value is stated in policy documents as those uploaded. An indicator is usually associated to a REST service (see below)

DIFFERENCE BETWEEN TARGET AND PRIORITY ACTION:

All targets and their related policies should be added to the RIS. However, only some targets is considered in relation to the Action Component. The targets related to the Action Component are considered Priority actions (see below).
